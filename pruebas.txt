Primero, se puede observar la ruta raíz ('/'). Esta es la puerta de entrada a la aplicación. Cuando se realiza una petición a la dirección principal del servidor —por ejemplo, accediendo a localhost:3000—, esta función se dispara y devuelve simplemente el texto 'Servidor activo'. En el desarrollo, esto se utiliza comúnmente como un chequeo rápido para confirmar que el servidor levantó de manera correcta.

Después, se puede observar una ruta estática en ('/saludo'). Esta ruta es muy directa. Si un cliente navega exactamente a esa dirección, el servidor siempre va a devolver el mensaje 'Hola comunidad'. Se le llama 'estática' porque su comportamiento y su respuesta son fijos; no cambian sin importar quién o cuándo se haga la petición.

Finalmente, se puede observar la ruta dinámica ('/mensaje/:nombre'). Aquí el comportamiento es diferente, ya que el :nombre con los dos puntos funciona como una variable dentro de la URL. Lo que hace Express es capturar cualquier valor que el usuario escriba en esa posición y lo guarda dentro de un objeto llamado req.params. Luego, el código toma ese valor y lo usa para construir una respuesta personalizada. Por ejemplo, si probamos la ruta escribiendo localhost:3000/mensaje/Joan, el servidor captura el nombre y responde dinámicamente con un 'Hola Joan'. Es la base para crear aplicaciones que interactúan de forma directa con los datos del usuario."
